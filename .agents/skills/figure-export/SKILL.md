## Position in this project

The figure is the product. The user makes figures to share — on Facebook,
in reports, in presentations, in academic papers. The export pipeline must
produce files that stand on their own without the app being open.

## Mandatory footer on every exported figure

Every exported figure carries, in the footer area:

1. **Data source(s)** with run identifier:
   - `Data: ECMWF Open Data (run 20260514 12z IFS)`
   - `Data: ECMWF ERA5 reanalysis (1991-2020 reference)`
   - `Data: JMA radar (valid 2026-05-14 12:30 JST)`
2. **License attribution**:
   - ECMWF: `CC-BY-4.0`
   - JMA: `出典: 気象庁ホームページ`
3. **App identifier and version**:
   - `Generated by AIseed Weather v0.1 (https://aiseed.dev)`
4. **Export timestamp** (UTC):
   - `Exported: 2026-05-14 03:45 UTC`

Implement once in `figures/footer.py`:

```python
def apply_footer(fig, *, data_source: str, run_id: str, license_text: str = "CC-BY-4.0"):
    text = (
        f"Data: {data_source} (run {run_id}). {license_text}\n"
        f"Generated by AIseed Weather v0.1 (https://aiseed.dev) — "
        f"Exported {datetime.utcnow():%Y-%m-%d %H:%M} UTC"
    )
    fig.text(
        0.99, 0.01, text,
        ha="right", va="bottom",
        fontsize=7, color="#555", family="sans-serif",
    )
```

The footer must be on the figure object itself, not added as a separate
overlay. If someone opens the PNG in any viewer, they see the attribution.

## Export formats

| Format | Use case | DPI | Notes |
|--------|----------|-----|-------|
| **PNG** | Default for sharing on web/SNS | 150 (default), 300 (HQ) | sRGB, with embedded metadata |
| **SVG** | Editing in Inkscape/Illustrator | vector | Useful for adding annotations |
| **PDF** | Print, report inclusion | vector | Embed fonts |
| **JPEG** | Only when explicitly requested | 150-300 | Avoid for charts with sharp lines |

PNG at 150 DPI is the default. The user can choose 300 DPI for higher
quality (slower export, larger file).

## Embedded metadata

PNG/SVG/PDF outputs embed metadata so the file is self-describing:

### PNG (using PIL or matplotlib's metadata kwarg)

```python
fig.savefig(
    path, dpi=dpi, format="png",
    metadata={
        "Title": title,
        "Author": "AIseed Weather user",
        "Software": f"AIseed Weather v{__version__}",
        "Source": data_source_long,
        "Copyright": license_text,
        "Description": f"Run {run_id}, variable {variable}, projection {projection}",
        "CreationTime": datetime.utcnow().isoformat(),
    },
)
```

### SVG

Matplotlib embeds metadata automatically via `metadata=` kwarg in the same
form.

### PDF

Pass the metadata dict to `savefig(format="pdf", metadata=...)`. matplotlib
maps it to PDF Info dict entries.

This metadata enables:
- Users discovering the file later can see what it shows
- Forensic verification of authorship/source
- Search tools indexing the file by topic

## Export workflow

```
User clicks Export
  → Modal asks: format (PNG default), DPI (150 default), filename
  → Re-render the figure at chosen DPI in a thread
  → Apply footer
  → Save with embedded metadata
  → Open the containing folder in the OS file manager
```

**Important**: do **not** reuse the on-screen figure object. Re-call the
figure-building function with export-quality parameters. The on-screen
figure may have low DPI or interactive elements that should not appear in
the export.

```python
async def export_figure(view_state, *, format: str, dpi: int, path: Path):
    # Re-render at export quality
    fig = await asyncio.to_thread(
        view_state.build_figure,
        export_dpi=dpi,
    )
    apply_footer(fig, **view_state.attribution())
    await asyncio.to_thread(
        fig.savefig,
        path, dpi=dpi, format=format,
        bbox_inches="tight",
        metadata=view_state.export_metadata(),
    )
    plt.close(fig)
```

## File naming

Default filename pattern:

```
aiseed-weather_<variable>_<run-id>_<projection>_<timestamp>.<ext>
```

Examples:
- `aiseed-weather_msl_20260514_12z_robinson_20260514T0345Z.png`
- `aiseed-weather_t2m-anomaly_era5_1991-2020_jp_20260514T0345Z.png`

Avoid spaces and special characters. Lowercase except for run-time HH (lowercase `z` is fine).

## Save location

Default directory: `~/Pictures/AIseed Weather/` (or platform equivalent via
`platformdirs.user_pictures_dir`).

User can change the default and pick per-export. Remember the last-used
directory in settings.

## Animation export (future)

When animation export is added:

- MP4 via `matplotlib.animation.FFMpegWriter` (requires `ffmpeg` from
  conda-forge)
- Frame rate 4-8 fps for synoptic time series (1 frame per 3-6 hours of
  forecast)
- Apply the footer to every frame, not just the first
- File size budget: aim for under 10 MB for a 10-day forecast loop at
  reasonable DPI

## Forbidden

- Exporting without the footer/attribution
- Reusing the on-screen figure object for high-DPI export
- Writing to disk on the UI thread (always `asyncio.to_thread`)
- Stripping metadata from re-saved files
- Defaulting to JPEG (lossy compression hurts contour lines)
- Hardcoding the save directory
- Silent overwrite of existing files (confirm with user)
